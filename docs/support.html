Frequently Asked Questions常见问题

What permissions does CopyHistory need?CopyHistory 需要哪些权限？

Accessibility permission is used only for automatic paste. Clipboard access is used to read new copied content so it can be stored locally. You can still browse history without Accessibility permission, but automatic paste will not work.

辅助功能权限只用于自动粘贴；剪贴板访问用于读取新复制的内容并保存到本机。没有辅助功能权限时，你仍然可以浏览历史记录，但无法自动粘贴。

How do I open clipboard history?如何打开剪贴板历史？

The default shortcut is Option + V. You can change it in Preferences, and you can optionally open the window by double-pressing a modifier key.

默认快捷键是 Option + V。你可以在偏好设置中修改，也可以启用双击修饰键打开窗口。

Where is my data stored?数据保存在哪里？

Released builds use App Sandbox by default. Data is stored under ~/Library/Containers/com.huagx.copyhistory/, including the SwiftData database and app preferences. Nothing is uploaded to a server.

发布版本默认启用 App Sandbox。数据保存在 ~/Library/Containers/com.huagx.copyhistory/ 下，包括 SwiftData 数据库和应用偏好设置。数据不会上传到服务器。

How do I clear history?如何清理历史记录？

Clear removes regular history entries while keeping pinned items, favorites, and manually listed items. Clear All removes every stored history entry after confirmation.

Clear 会清理普通历史记录，但保留置顶、收藏和手动加入列表的项目；Clear All 会在确认后删除所有历史记录。

What content types are supported?支持哪些内容类型？

CopyHistory supports text, images, files, and other supported pasteboard types. Search and preview behavior depends on the content type.

CopyHistory 支持文本、图片、文件以及其他受支持的剪贴板类型。搜索和预览能力会根据内容类型有所不同。
